Detailed Description

Includes, defines and function prototypes for interrupt based I2C communication implementation.

Author:
Orestis Zachariadis

Instructions:

  • Include this file in i2c_api.c and disable i2c_clear_SI() and i2c_wait_SI() in i2c_api.c.
  • Call I2C?_queue_create()
  • Make a thread that will use the functions of the MBED I2C api (http://mbed.org/handbook/I2C) to communicate with a sensor. Api is exactly the same. This way this thread will be put into WAITING state while waiting for SI interrupt and leave the processor free for other tasks.
Note:
Due to MBED Library bug, i2c_write() and i2c_read() in i2c_api.c were changed in order to support a repeated start i2c_read after an i2c_write call.

Instructions if you want to use repeated start:
  • Delete original i2c_api.c and use i2c_api2.c in LibMods folder instead.
  • If you set "repeated" argument to true in I2C::write() the following I2C::read() must also have "repeated" argument set to true.
  • An I2C transaction is not allowed to start with I2C::read() with "repeated" argument set to true.
Attention:
As of now only the sequence write -> repeated start -> read is implemented.
Note:
Changes in original i2c_api.c are marked with "CHANGED MBED LIBRARY HERE" in comments.

void I2C0_IRQHandler ( )

I2C0 Interrupt Handler.

Disables interrupts

Note:
We must disable interrupts because we don't clean the SI flag yet. Otherwise the ISR would retrigger infinitely.

Then adds an element to I2C0_queue

Note:
Doesn't matter what element, we use the queue only as signaling mechanism. We can't use RTOS signals because they can notify only specific threads.

void i2c_clear_SI ( i2c_t *  obj) [inline]

Clears the Serial Interrupt (SI)

Clears SI and enables interrupts

Note:
If we didn't clear pending interrupt, IRQHandler would trigger, even though interrupt flag would be clear (for I2C SI=0 means status 0xF8).
Interrupt flag must be cleared BEFORE clearing the pending interrupt, or the pending interrupt would be set again.
Parameters:
objpointer to i2c_t struct of the corresponding I2c peripheral

int i2c_wait_SI ( i2c_t *  obj)

Wait for Serial Interrupt (SI)

Waits an interrupt of the corresponding I2c peripheral to fill the queue so it can continue execution. While it waits the thread is put into WAITING state, so the CPU is free for other tasks.

Parameters:
objpointer to i2c_t struct of the corresponding I2c peripheral
Returns:
0 if successful
