Detailed Description

MBED i2c_api.c with modifications and fixes.

Note:
Due to MBED Library bug, i2c_write() and i2c_read() in i2c_api.c were changed in order to support a repeated start i2c_read after an i2c_write call.

Instructions if you want to use repeated start:
  • Delete original i2c_api.c and use i2c_api2.c in LibMods folder instead.
  • If you set "repeated" argument to true in I2C::write() the following I2C::read() must also have "repeated" argument set to true.
  • An I2C transaction is not allowed to start with I2C::read() with "repeated" argument set to true.
Attention:
As of now only the sequence write -> repeated start -> read is implemented.
Note:
Changes in original i2c_api.c are marked with "CHANGED MBED LIBRARY HERE" in comments.
